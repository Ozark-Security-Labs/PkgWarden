# Support

Use GitHub Issues for bugs, feature requests, rule requests, documentation gaps,
framework support, and false positives or false negatives.

Before opening an issue, include:

- PROJECT_NAME version or commit.
- Operating system and runtime/toolchain version.
- Minimal reproduction or fixture.
- Relevant configuration excerpt.
- Expected and actual output.

For security-sensitive reports, follow [SECURITY.md](SECURITY.md) instead of
opening a public issue.
